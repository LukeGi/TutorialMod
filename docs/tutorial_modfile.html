This Website is all about showing you how to make a Minecraft Mod with the Minecraft Forge MDK.

Making Your @Mod File

@Mod

FML (Forge Mod Loader), now a component of Minecraft Forge, uses the @Mod annotation to locate the mods when loading the game. It takes the following arguments:
String modid() **
String name() default "" *
String version() default "" *
String dependencies() default ""
boolean useMetadata() default false
boolean clientSideOnly() default false
boolean serverSideOnly() default false
String acceptedMinecraftVersions() default ""
String acceptableRemoteVersions() default ""
String acceptableSaveVersions() default ""
Stirng certificateFingerprint() default ""
String modLanguage() defualt "java"
String modLanguageAdapter() default ""
boolean canBeDeactivated() default false
String guiFactory() default ""
String updateJSON() default ""
CustomProperty[] customProperties() default {}
 ** use is required
* use is recommended

When I make my mods I generally use the following as a bare minimum, where ModInfo.class is a reference class:
@Mod(modid = ModInfo.MOD_ID, name = ModInfo.MOD_NAME, version = ModInfo.VERSION)
public class MyMod
{
// modfilestuff
}
Right... now we have the @Mod annotation out of the way, we can move on to the contents of the Mod file. This is a very subjective matter; some people need some things where others don't. So here are all the things that you might need... in order of likelyness of requirement.


@Mod.EventHandler-s


FMLPreInitializationEvent

There are a few things that you can do in this event. the recommended ones are as follows:
*This is a slightly modified version of the documentation copied directly from the documentation on the FMLPreInitializationEvent class*
> Setup your logging e.getModLog()
> Load any configuration data you might have e.getSuggestedConfigurationFile()
> Search for a version.properties file and load it e.getVersionProperties()
> Configure your ModMetadata programmatically e.getModMetadata()
> Register your blocks and items with the net.minecraftforge.fml.common.registry.GameRegistry
> Discover parts of your mod by using annotation search e.getAsmData()
